# VCF 联系人解析与修正

对 VCF (vCard) 联系人文件进行解析、显示，并支持自动修正：加国际区号、联系人去重与号码合并、重复姓名修正。**不指定功能选项时，默认自动修复并写入日志。**

## 功能概览

- 解析并显示联系人（支持 UTF-8 / QUOTED-PRINTABLE、续行）。
- 自动添加国际区号（+86 / +852 / +853），并保留原号码。
- 联系人去重与号码合并（同名合并为一条，号码合并）。
- 重复姓名修正（如「王三王三」→「王三」，≤2 字姓名不去重）。
- 变更日志：控制台 + 可选写入 `vcf_fix_YYYYMMDD_HHMMSS.log`。

## 功能选项

| 选项 | 说明 | 默认 |
|------|------|------|
| 自动添加国际区号 | 为号码补充 +86/852/853，保留原号 | 开启 |
| 存储变更日志 | 将变更写入本地 log 文件 | 开启 |
| 自动修正姓名 | 重复姓名去重（如王三王三→王三） | 开启 |
| 自动合并号码 | 同名联系人合并，号码合并到同一人 | 开启 |
| 打印所有号码 | 仅展示全部联系人及号码，不写回 | 查询模式 |
| 按姓名查号码 | 显示指定姓名下的全部号码 | 查询模式 |
| 按号码查联系人 | 显示包含指定号码的全部联系人 | 查询模式 |
| 仅姓名无号码 | 最后输出无任何号码的联系人列表 | 自动输出 |
| 删除无号码联系人 | 不将无号码联系人写入输出 VCF | `--remove-no-tel` |

指定 `--list`、`--name`、`--number` 时为**仅查询模式**：只做展示，不修正、不写回 VCF。

## 环境

- Python 3.6+
- 仅使用标准库（无第三方依赖）

## 用法

```bash
# 默认：自动修复并写日志，输出到 输入名_fixed.vcf
python3 vcf_fix.py
python3 vcf_fix.py /path/to/contacts.vcf -o /path/to/output.vcf

# 关闭某项自动修复
python3 vcf_fix.py --no-country-code    # 不添加国际区号
python3 vcf_fix.py --no-log             # 不写日志文件（仅控制台）
python3 vcf_fix.py --no-fix-name        # 不修正重复姓名
python3 vcf_fix.py --no-merge           # 不合并同名联系人

# 仅查询/展示（不写回文件）
python3 vcf_fix.py --list               # 打印显示所有联系人及号码
python3 vcf_fix.py --name "张三"        # 显示该姓名下的全部号码
python3 vcf_fix.py --number "13800138000"  # 显示包含该号码的全部联系人

# 无号码联系人：修复结束后会列出「仅姓名无号码」的联系人；加下方选项则从输出文件中删除他们
python3 vcf_fix.py --remove-no-tel        # 删除无号码联系人（不写入输出 VCF）
```

## 输出格式与兼容性（vCard 2.1 / RFC 2426）

- **N、FN、ADR**：输出使用规范形式（N/FN 为 `N;CHARSET=UTF-8:;姓名;;;`、`FN;CHARSET=UTF-8:姓名`，无 ENCODING），折行与原文件一致。姓名修正（如 张三张三→张三）与「姓名+数字」优先仅用于合并与显示。
- **TEL**：由程序按合并/区号结果生成，按 RFC 2426 **75 字符**折行；其余属性（PHOTO、X-GROUP 等）原样输出。
- **属性顺序**：与源文件一致（N → FN → TEL… → 其它 → END）。
- **换行**：CRLF（`\r\n`）。

## 输出

- **控制台**：解析数量、修正前/后联系人列表、区号/合并/姓名修正说明、**仅姓名无号码的联系人列表**、修复完成汇总、输出路径。
- **日志文件**：与上述一致写入 `vcf_fix_YYYYMMDD_HHMMSS.log`（可用 `--no-log` 关闭）。

## 文件说明

- `vcf_fix.py`：主程序。
- `*_fixed.vcf`：默认修正输出（可由 `-o` 指定）。
- `vcf_fix_*.log`：每次运行生成的日志（`--no-log` 时不生成）。
